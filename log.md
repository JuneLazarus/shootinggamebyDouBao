# 《彩铅线稿 FPS》开发日志

单文件网页 FPS，纯 Three.js 实现无打包依赖，本地打开即玩，彩铅手绘美术风格

## 项目概览

| 项 | 说明 |
| ---- | ---- |
| 技术栈 | Three.js 核心库 + 原生 Canvas2D 后处理 + Web Audio 生成音效 |
| 美术定位 | 铅笔草稿 + 彩铅薄涂质感，米黄纸张肌理，微色差模拟手绘套色误差 |
| 核心玩法 | 第一人称射击 + 跑酷跳跃 + 随机生成地图，低门槛快节奏 |
| 包体大小 | 单 HTML 文件～30KB，所有资源程序化生成无外部依赖 |

## 版本迭代记录

### v0.1 线稿原型版 | 初始核心搭建

开发目标：跑通最基础 FPS 循环
✅ 完成功能：

第一人称相机 + WASD 移动 + 鼠标视角控制，浏览器指针锁定
手枪基础射击逻辑：12 发弹匣 / 36 发备弹，R 键换弹，后坐力反馈
手绘线稿渲染：白底 + 黑色轮廓，每帧线条微抖动模拟铅笔手绘质感
基础敌人 AI：随机生成后直线追击玩家，近身造成伤害，受击变色反馈
AABB 碰撞检测，围墙壁垒 + 随机方块障碍物
HP 条、分数、弹药数 UI，十字准星
🐛 初始遗留问题：
无跳跃无高低差，全平面地形
无空中移动控制，碰撞逻辑生硬容易卡
美术风格单一，纯黑白辨识度低

### v0.2 垂直地形版 | 加入跳跃与落差

开发目标：增加垂直维度玩法
✅ 完成功能：

重力 + 空格跳跃系统，物理模拟落地反馈
高低差地形体系：1 米高台、0.5 米台阶、垫脚方块、多层级阶梯
调整敌人模型比例，站立时头部高度与玩家摄像机完全平齐（平视刚好对视）
自动台阶爬升：0.5 米以下矮台阶无需按空格，走上去自动抬升
🔧 踩坑修复：
第一版跳跃逻辑存在严重 bug：空中无法平移、跳台阶时被侧面碰撞卡住，前后重写 3 次水平碰撞逻辑才解决卡墙问题
给敌人同步增加重力和台阶爬升能力，不会卡在矮台阶下摸不到高台上的玩家

### v0.3 彩铅美术升级 | 风格定型

开发目标：从黑白线稿升级为彩铅手绘质感
✅ 完成功能：

米黄色彩铅纸底色，程序化生成纸张纤维、淡彩杂点肌理
物体统一使用低饱和高明度淡彩薄涂，按功能分配基础色
微色差后处理：RGB 通道极轻微偏移模拟彩铅叠色错位，动态颗粒噪点模拟铅芯质感
全 UI 配色改版，适配暖调手绘风格
💥 重大踩坑：
最初用 Three.js 官方后处理扩展实现色差效果，因 ES 模块加载限制，本地直接打开 HTML 会因跨域问题白屏
解决方案：抛弃所有第三方扩展依赖，采用「隐藏 WebGL 画布渲染 + Canvas2D 手写后处理管线」的方案，最终仅依赖 Three.js 核心库，本地打开 100% 稳定运行

### v0.4 大地图版本 | 随机生成与补给

开发目标：扩大游玩空间，完善收集循环
✅ 完成功能：

地图从 50x50 米扩建为 80x80 米，每次重开随机生成地形形状 / 高度 / 位置，高于 1 米的高台自动生成垫脚块保证可到达
明黄色弹药补给，全地图 + 高台上分布，拾取 + 24 发备弹，10 秒自动刷新
彻底修复空中移动 bug：跳跃过程中可自由前后左右平移控制落点，碰撞判定不锁水平输入
功能色体系落地：矮台阶 / 高台 / 围墙 / 敌人 / 补给用不同淡彩色区分，辨识度拉满
🔧 优化：
碰撞精度调整，台阶爬升增加 5cm 容错，不会因浮点误差卡脚

### v0.5 射击手感迭代 | 瞄准与爆头

开发目标：让射击有深度不无聊
✅ 完成功能：

动态腰射散布：站定散布最小，移动 / 疾跑 / 跳跃时散布逐级增大，准星同步动态张开，直观显示当前弹道范围
右键机瞄系统：枪械平滑抬到视野中心，FOV 收窄到 65 度、移速降低 20%、鼠标灵敏度降低 25%，弹道散布缩小到 0.2 度几乎精准
爆头判定：敌人头部（1.7 米高球体区域）命中 3 倍伤害，手枪爆头一枪秒杀普通敌人，带专属高音命中音效和飘字提示
朱红色十字血包补给，拾取回复 30HP，15 秒刷新，高台上作为探索奖励
🔧 调整：
最初瞄准过渡时间设为 0.3 秒太拖沓，直接提速到 0.1 秒快速响应，兼顾顺滑跟手
取消子弹重力下坠，手绘风格走爽快射击路线，指哪打哪

### v0.6 动作层与音效 | 沉浸感补全

开发目标：丰富动作反馈和听觉体验
✅ 完成功能：

Shift 疾跑：1.25 倍移动速度，FOV 拉宽到 80 度，持续消耗体力，带低通滤波风声效果
体力条系统：疾跑每秒耗 28 点、每次跳跃耗 15 点，落地每秒回 18 点；体力耗尽时无法疾跑、跳跃高度减半
全套交互音效（全部 Web Audio 原生生成，零外部音频文件）：射击、换弹拉栓 / 上膛、跳跃、轻重落地、弹药 / 血包拾取、玩家受击、爆头、空仓咔哒、疾跑风声
BGM 接口：支持将本地音乐命名为bgm.mp3放在同目录自动循环播放，无音乐文件时自动生成极轻低频环境白噪避免完全静音
（临时测试）近战木棒：左键轻击 / 右键蓄力重击，带击退效果，后续因玩法聚焦移除

### v0.7 稳定版 | 精简体验 + Bug 清零

开发目标：砍掉冗余内容，修完影响体验的核心 bug
✅ 完成内容：

移除临时加入的近战系统，回归纯手枪射击核心玩法，删除滚轮切枪等冗余操作，操作逻辑简化为「移动 / 跳跃 / 射击 / 瞄准 / 换弹」
音量调节控件从游戏内常驻 HUD 移到菜单界面（开始 / 暂停 / 结算页），游玩时界面更干净，暂停时随时可调
彻底修复手枪模型消失 bug：原走路晃动逻辑用坐标累加偏移，长时间移动会出现数值溢出导致模型跑出视野，重写为「目标位置插值」逻辑，所有状态（瞄准 / 换弹 / 落地 / 走路晃动）统一计算最终坐标，武器位置 100% 稳定
调整所有音效音量平衡，避免射击 / 爆头音刺耳
准星动画优化，散布大小过渡更顺滑无跳变
🐛 当前遗留小问题（不影响核心游玩）：
极端贴边角度下敌人偶尔会轻微穿台阶
随机地形极低概率生成无垫脚的孤立高台，后续优化生成规则
开发心得

稳定性优先于效果：一开始为了实现色差效果堆第三方扩展，结果本地打开直接白屏，最后花一晚上手写 Canvas 后处理虽然麻烦，但换来了单文件随便传都能跑的兼容性，网页游戏这点最重要
手感是磨出来的：跳跃高度、台阶容错、瞄准速度、散布大小、后坐力幅度这些数值没有标准答案，前前后后调了十几次才达到 “顺” 的手感
做减法比加内容难：一开始加近战、切枪、一堆花活，最后发现反而让操作变复杂，删掉冗余功能后体验反而更流畅，聚焦 “跑 + 跳 + 射” 的核心爽点就够了
